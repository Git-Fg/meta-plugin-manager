## MCP Server Contracts

### MCP: server-name

#### Tool: tool_name

**Purpose**: What this tool does

**Input Schema**:
```json
{
  "type": "object",
  "properties": {
    "param_name": {
      "type": "string",
      "description": "Parameter description"
    }
  },
  "required": ["param_name"]
}
```

**Output Schema**:
```json
{
  "type": "object",
  "properties": {
    "result": {
      "type": "string",
      "description": "The result"
    }
  }
}
```

**Error Cases**:
- `InvalidInput`: Raised when input validation fails
- `NotFound`: Raised when requested resource doesn't exist

## Integration Contracts

### Flow: flow-name

**Components**: `[component-a, component-b, component-c]`

**Contract**:
1. `component-a` receives user input
2. `component-a` emits `event.next`
3. `component-b` handles `event.next`
4. `component-b` calls `component-c` tool
5. `component-c` returns result
6. Result flows back through chain to user

**Success Criteria**:
- All components execute in order
- No data loss between components
- Error at any stage propagates correctly
